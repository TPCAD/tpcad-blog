# Regular Express Reference

## 元字符

元字符是具有特殊含义的字符，如果要表达元字符本义，需要使用`\`转义。

### . 任意字符

`.`表示除换行符以外的任意单个字符。

### 字符集

#### [] 自定义字符集

`[]`表示匹配括号中的任意一个字符。

当`^`位于`[]`中且位于开头时，表示匹配除`[]`内字符之外的任意一个字符。`-`表示连续范围，如`[a-z]`表示所有小写英文字母，`[0-9]`表示所有阿拉伯数字。

位于`[]`中的特殊字符不需要转义。

#### \d \D 数字字符集

`\d`表示匹配任意一个阿拉伯数字，等价于`[0-9]`，`\D`表示匹配任意一个非阿拉伯数字，等价于`[^0-9]`。

#### \w \W 单词字符集

`\w`表示匹配任意一个单词字符（大小写英文字母、阿拉伯数字、下划线），等价于`[a-zA-Z0-9_]`，`\W`表示匹配任意一个非单词字符，等价于`[^a-zA-Z0-9]`。

#### \s \S 空白字符集

`\s`表示匹配任意一个空白字符（空格、制表符、换行符等），`\S` 表示匹配任意一个非空白字符。

### 量词

正则表达式默认是「贪婪匹配」，即尽可能多地匹配字符。如`<.*>`会匹配整个`<p>Hello World</p>`。在量词后加上`?`即可开启非贪婪匹配。如`<.*?>`只会匹配`<p>`。

#### * 匹配零次或多次

`*`表示匹配其前面的子表达式零次或多次。

#### `+` 匹配一次或多次

`+`表示匹配其前面的子表达式零次或多次。

#### `?` 匹配零次或一次

`?`表示匹配其前面的子表达式一次或零次。

#### {n} 匹配刚好 n 次

`{n}`表示匹配其前面的子表达式刚好 n 次。

#### {n,} 匹配至少 n 次

`{n}`表示匹配其前面的子表达式至少 n 次。

#### {n,m} 匹配 n 到 m 次

`{n,m}`表示匹配其前面的子表达式 \[n,m\] 次。

### 边界

#### ^ 字符开始

`^`表示匹配以其后方子表达式开头的字符串。

#### \$

`$`表示匹配以其前方子表达式结束的字符串。

#### \b \B 单词边界

`\b`表示匹配单词边界，`\B`表示匹配非单词边界。

```language
See the Third Party Plugins wiki page for a list of available backends.

For information on how to create a new backend, see the Backends for Developers chapter.
```

`for`会匹配单独的`for`和`information`中的`for`。`for\b`只会匹配单独的`for`，而`for\B`则只会匹配`information`中的`for`。

### () 分组

`()`将其中的多个字符视为一个整体单元。单元可像普通字符一样应用量词。

例如`(ab)+`会匹配`abc`、`ababd`、`abababa`，但不会匹配单独的`a`和`b`。

默认情况下，分组会被分配一个从 1 开始的索引号，使用索引号可以引用对应的分组，这种分组成为「捕获分组」。

例如，`<([a-z][a-z\d]*)\b[^<]*>.*?</\1>`用于匹配 HTML 标签，其中的`\1`表示引用前面的捕获分组`([a-z][a-z\d]*)`，确保标签一致。

若不想捕获分组可在表达式前加上`?:`，如`(?:\d{4})-(\d{2})-(\d{2})`不会捕获分组`(\d{4})`。

### | 分支

`|`表示分支。如`apple|pen`会匹配`apple`或`pen`。`[]`相当于单字符版的`(|)`，如`[Pp]`等价于`(P|p)`。

## 断言

断言用于限定匹配结果的位置，即匹配结果的前面或后面可以或不可以是某种模式。据此，可以分类出四种断言类型：

- 正向先行断言
- 负向先行断言
- 正向后行断言
- 负向后行断言

**正向先行断言**表示匹配结果的前面（右边）必须符合模式，其语法为`(?=pattern)`。

```language
the app is 100 dollars, the pen is 1 dollar, the dollar is 100.
```

`\b\d+\b(?=\s?dollars?)`表示匹配单词`dollar`或`dollars`前的数字。除了最后一个 100 外，其他数字都符合条件。

**负向先行断言**表示匹配结果的前面（右边）必须不符合模式，其语法为`(?!pattern)`。

`\b\d+\b(?!\s?dollars?)`表示匹配后面没有单词`dollar`或`dollars`前的数字。除了最后一个 100 外，其他数字都不符合条件。

**正向后行断言**与**负向后行断言**分别表示匹配结果的后面（左边）必须符合或不符合模式，语法分别为`(?<=patter)`和`(?<!patter)`。
